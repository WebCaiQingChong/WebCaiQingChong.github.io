啃一啃犀牛书--初始

  • javascript

犀牛书已经买了很久了，而且已经脏的不行，但是我还是没有真正的读过他，本着夯实基础的心理，系统的来一次(ps:实际上花了钱不能浪费啊，犀牛书好贵的)

声明

本次系列，除了名词定义等，其他均为个人理解，如有出错，欢迎拍砖。

这里记录的是一个学习的过程，如果其中的万分之一可能帮到你，那对我来说就值了。
虽然标题是啃犀牛书，但是对于里面的一些理解，我依旧会查阅多方资料，书籍来进行诠释，希望可以做的更加详尽。
犀牛书的厚度想必大家也都知道，这个系列估计要持续挺长的时间，我希望自己可以坚持下来，也希望大家可以进行拍砖，以鼓励我继续下去。

说在前面

因为现阶段，我对于基础的掌握是在不尽如人意，匆匆翻了几眼书，也只会有临时的记忆，这并不是我的本意。
首先，我要承认学习是一个痛苦的过程，我不太相信既可以快乐，又可以学到东西(ps:css貌似是这样的，但是里面也有让你痛苦的东西，比如说兼容？)
其次，学习的过程应该会是漫长的，对于某一类的语言的掌握，肯定不是一蹴而就的，是需要时间来积累的(ps:如果你真的是天才，那恭喜你，但是我相信我们都是平凡的人，都要通过自己的努力去实现。)
最后，我得承认，你把犀牛书看完了，并不代表你会js，或者说掌握了js。这就跟努力不一定成功，不努力一定不成功是一个道理。

内容概要

博客建立之后的几篇文章，现在读来真心觉得没有什么营养，基本上是摘抄之类的。此系列文章也不会采用这种方法，不会因为需要内容而去堆砌内容，力求做到每篇文章都是精心准备的。
希望可以做到，让你看完以后，并没有觉得是浪费时间。(ps:只能说希望，毕竟现在的技术还很差)

即便是书或者其他的资料，并不一定都100%是正确的，希望大家在看过之后，可以质疑出来，质疑的过程肯定是一个提高的过程。
所有新的技术或者事务的产生，我相信应该都是在质疑中产生的。

期望

我很想一天分享一篇，但是没办法，不会有那么多的时间，尽量控制频率在三天以内(包括三天)整理出一份，因为js的知识点有时候会很碎，三天整理出来的文章，内容应该不会太多，
但是现在都比较倡导碎片化的记忆，利用碎片化的时间。我肯定没心情写长篇大论，你肯定也没有时间去看。

希望可以多进行交流，博客已开放评论区，不过需要github账号进行登录，因使用的gitment目前还是有一些缺陷的，如果出现问题，可以直接点击issue page，直接跳至issue进行交流，
github跟我邮箱绑定，我可以及时的进行响应。

如果按照犀牛书的章节来说，每次一个章节，肯定很快就可以搞定了，但是我觉得其中有些章节，真的不是一篇简单的文章就可以全部囊括。

如果你已经犀牛书啃了不知道多少遍了，我也请你可以耐心的看下去，当做是给其他观看者提出问题，提出质疑。

文中的错误，请务必指出，否则，这个系列很可能就是在误人子弟了。

如果觉得我写的还能看看，欢迎分享~

It’s time to study~~~~
