美团移动端-巧用伪类布局

  • css

今天在小程序前端FED的文章里面看到了关于每段移动端选择城市的一个布局，忽然觉得这个布局其实挺巧妙的。

布局如下:

在查看样式的时候发现，最后一个空格里面是没有元素的。下面自己写了两种，加上美团的视线方案。(ps:挺久不切图了，代码很糟。)

原图地址:美团网
效果图就不再贴了，首先我的两种方案也可以实现，但是第一种方案有瑕疵，右下角会有空白，第二种方案并不是很适合移动端布局。
两种方案的通病就是为了占位创建了多余的元素，从语义化来讲，这并不合理。创建空元素来占行。

美团方案解析：
外框的用外层div.mt进行创建，中间的横线由li进行创建，竖线使用伪类。

美团的方案利用伪类进行布局，代码上也算优雅，好处是它根本不需要管下面到底多少个元素，我们的还要人为的填充。
这个代码本身也有缺陷，因为你可能也看出来了，伪类值可以用两个，创建出来的元素只能有两条线，也就是说最多只能画出四条线，最多进行五栏的布局。
但是从当前需求来将，确实是一种比较可行的方案。

其他的小伙伴有兴趣也可以自己研究一套更为合适的方案，首先，我们尽量用js去控制，那就有点得不偿失了。
